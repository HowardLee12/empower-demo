# 賽事管理後台 — 架構設計

## 資料層級關係

```
Organization (球隊)          例：台北黑熊隊、板橋神鷹隊
  ├── Player (球員)          例：王小明（姓名、背號、生日）→ 屬於黑熊隊
  └── Squad (小隊)           例：黑熊 U12、黑熊 U15
        └── sb_squad_players  多對多：哪些球員在這個小隊（從同球隊的球員中挑選）

League (賽事)                例：2026 Final、Galaxy Cup
  ├── sb_league_squads       哪些小隊參加此賽事
  ├── sb_league_rosters      參賽名單（特定賽事中某小隊的球員 + 背號）
  └── Game (比賽)            例：黑熊 U12 vs 神鷹 U12
        └── PlayerStats      每場球員數據
```

## 管理後台四個 Tab 的職責

### Tab 1：球員管理
**職責：球員 CRUD + 指定所屬球隊**

操作流程：
1. 選擇球隊（左欄）
2. 查看/新增/編輯/刪除球員（中欄）
3. 點選球員 → 指定他加入哪些小隊（右欄）

新增球員欄位：
- 姓名（必填）
- 背號
- 生日（必填）
- 所屬球隊（必填，選 Organization）

重點：
- 球員新增時一定要指定球隊
- 一個球員只屬於一個球隊
- 一個球員可以在同球隊的多個小隊（例如同時在 U12 和 U15）
- **這裡不新增球隊/小隊**，球隊和小隊在「隊伍管理」建立

### Tab 2：隊伍管理
**職責：球隊 + 小隊的 CRUD**

操作流程：
1. 新增/編輯/刪除球隊（左欄）
2. 選擇球隊 → 新增/編輯/刪除小隊（中欄）
3. 選擇小隊 → 查看該小隊的球員名單（右欄，唯讀展示）

新增球隊欄位：
- 名稱（例：台北黑熊隊）
- 簡稱（例：黑熊）

新增小隊欄位：
- 名稱（例：黑熊 U12）
- 分組（例：U12）

重點：
- **不在這裡新增/管理球員**（球員在「球員管理」處理）
- 右欄只展示小隊目前有哪些球員（唯讀）
- 如果要調整球員分配，引導到「球員管理」tab

### Tab 3：賽事管理
**職責：賽事 CRUD + 指定參賽小隊 + 設定參賽名單**

操作流程：
1. 新增/編輯/刪除賽事（左欄）
2. 選擇賽事 → 新增/移除參賽小隊（中欄）
3. 選擇小隊 → 設定此賽事的參賽名單（右欄，從小隊球員中挑選）

新增賽事欄位：
- 名稱（例：2026 Final）
- 類型（例：正式錦標賽）
- 賽季（例：2026 Q1）

重點：
- 參賽名單（sb_league_rosters）是針對特定賽事的
- 同一個球員在不同賽事可以有不同背號
- 參賽名單的球員來源是該小隊的球員（sb_squad_players）

### Tab 4：賽程管理
**職責：排定比賽 + 開始記錄**

操作流程：
1. 新增比賽：選日期/時間/地點/賽事/主客隊
2. 待開始的比賽列表 → 點「開始記錄」進入紀錄台
3. 已結束的比賽列表 → 點「數據」查看 Box Score

新增比賽欄位：
- 日期
- 時間
- 地點
- 賽事（選 League）
- 主隊（選 Squad）
- 客隊（選 Squad）

重點：
- 比賽必須指定賽事
- 開始記錄時，球員名單自動從 sb_league_rosters 載入
- 如果沒有設定 roster，fallback 到小隊全部球員

## 操作步驟摘要

```
Step 1: 隊伍管理 → 建立球隊 → 建立小隊
Step 2: 球員管理 → 建立球員（指定球隊）→ 分配到小隊
Step 3: 賽事管理 → 建立賽事 → 加入參賽小隊 → 設定參賽名單
Step 4: 賽程管理 → 排定比賽（選賽事+主客隊）→ 開始記錄
Step 5: 紀錄台 → 記錄比賽 → 結束儲存
Step 6: 前台自動顯示結果
```

## 資料表對應

| 管理 Tab | 主要操作的資料表 |
|---------|---------------|
| 球員管理 | sb_players（CRUD）、sb_squad_players（分配小隊） |
| 隊伍管理 | sb_organizations（CRUD）、sb_squads（CRUD） |
| 賽事管理 | sb_leagues（CRUD）、sb_league_squads（參賽小隊）、sb_league_rosters（參賽名單） |
| 賽程管理 | sb_games（CRUD）→ sb_player_stats（紀錄台產生） |
